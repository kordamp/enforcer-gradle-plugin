Package: enforcer.rules.internal

[Groovy] Class ClassesWithSameName

    • enforcer.rules.internal.ClassesWithSameName

    • Represents one or more class files that have the same exact name. In this case the class name is a relative, file system path to the class file. For example: org/apache/maven/Stuff.class Example of how we can have two of the same class: - mockito-core-1.9.5.jar contains org/mockito/Mockito.class - mockito-all-1.9.5.jar contains org/mockito/Mockito.class With that example you're not supposed to have both on the classpath. Typically you'd choose the maven way (mockito-core) or the convenient-for-non-maven-users way (mockito-all) but not both. Adapted from org.apache.maven.plugins.enforcer.ClassesWithSameName.
